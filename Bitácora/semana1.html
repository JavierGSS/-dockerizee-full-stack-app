1. Variables

Las variables pueden ser definidas como los nombres que le damos a los datos que usamos para construir y ejecutar un programa. Hay diferentes tipos de datos; hay cadenas, números, booleanos, pero también hay arreglos, objetos, fechas, y algunos otros tipos más. Se denomina el valor de una variable a la cosa o dato nombrado por ella (un número, un arreglo, etc.), y decimos que hemos definido una variable cuando le hemos asociamos a esa variable algún valor. Para definir una variable en JS se utilizan ciertas palabras-clave, las palabras let, var y const. En el siguiente snippet se ilustra cómo se pueden definir tres variables diferentes, uno, persona y arreglo, y asignárseles tres tipos de datos diferentes (un número, un objeto y un arreglo, respectivamente): 
                    let uno = 1;

                    var persona = {
                        nombre: 'emilia',
                        apellido: 'guzmán',
                        edad: 35
                    };

                    const arreglo = [0, 'A', 2, [1]];
Las variables tienen un alcance, que puede definirse como el fragmento del programa en el cual las variables existen o están definidas. Las variables introducidas con const y var, así como las que fungen como argumentos o parámetros de función, existen únicamente dentro de las funciones que las movilizan. Las variables definidas con let tienen un alcance de bloque, esto es, definido estrictamente por las llaves ({ }) en las que se insertan. Por último, hay variables globales, que existen dentro de todo el programa que las emplea y se definen por fuera de cualquier bloque definido por llaves. En el snippet que incluyo abajo se muestra una variable, global, cuyo alcance es global, y otra, local, cuyo alcance es de función: 
                    var global = [0, 1, 2];

                    function cuadrado(x) {
                        let local = x * x;
                        return local;
                    }
Cuando dos variables refieren a un mismo tipo de dato primitivo (i.e., a números, cadenas, booleanos, undefined, null y symbol), sus valores son independientes. Por tanto, modificar el valor de una de ellas no altera el valor de la segunda. No sucede lo mismo cuando refieren a un tipo de referencia, como un arreglo o un objeto. En este caso, ambas variables refieren a lo mismo (y no a diferentes copias de lo mismo), y por consiguiente modificar el valor de alguna de ellas implica una modificar el valor de la segunda.

2. Heap y stack

El stack es una estructura estática de datos, mientras que el heap es una estructura dinámica de datos. En el stack existe por omisión un marco global que aloja a las variables globales, esto es, a los nombres asociados a la info que deseamos tener a la mano a través de todo el programa. Ahora bien, dónde se aloja esta info depende del tipo de dato asociado a ella. Si se asocia a un número, a una cadena o a un Booleano, entonces su valor se guarda directamente en el stack. Si el tipo de dato asociado es una función, por ejemplo, o un arreglo, entonces su valor no se copia en el stack, sino en el heap, y la relación entre variable (que se registra en el stack) y su valor (contenido en el heap) se conoce como "referencia". Los arreglos y los objetos (entre los cuales se cuentan las funciones), pues, no viven, como los números o las cadenas, en el stack, sino que viven en el heap, y existen solamente en tanto haya algo en el stack (una variable) que haga referencia a ellos.

A medida en que el programa corre y se van ejecutando funciones, se van creando nuevos marcos en el stack asociados a la aplicación de estas funciones a argumentos concretos. Estos marcos sólo existen durante la ejecución de la función. En una imagen, el stack es el escenario donde se desenvuelve toda la acción; el heap, en contraparte, es donde se almacenan todos los elementos que tienen un papel en la obra.
